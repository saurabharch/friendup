Dormant

Introduction

Have you ever wished that your applications could talk together? Not literally, ofcourse, but passing messages to each other, using each other's functionality? Well, with Dormant, this is now possible in a radical way.

According to Webster's dictionary, Dormant means:

  • not doing anything at this time : not active but able to become active

This is exactly what Dormant applications in FriendUP does. They idle, and wake up when different events occur. Additionally, Dormant applications can leverage the functionality in other applications, as well as the inherent Workspace functions that also expose Dormant functionality.

Use cases

  • When using applications, there are often features that are missed that are available in other applications. With Dormant, one can access all application functions through a single, simple and system wide interface.
  • When programming an application, the application itself can access its own functionality through Dormant, thereby building new functions at a high level - either through Javascript or through FriendDOS scripting.
  • Applications that use Dormant can be remote controlled by authorized administrators in your network. This allows new support schemes and opportunities.
  • Programmers can get easy access to unit testing, allowing scripting of manual tasks.

Getting started

To get started using Dormant, you can start by considering the built-in system-wide Dormant functions available in the System: volume. This volume is available to anyone using the FriendUP Workspace.

In the System: volume, you will find a directory called Functions. This directory contains a set of functions with which you can control the FriendUP desktop functionality, like mounting volumes, opening views and screen surfaces. You can test these functions by doubleclicking on their icons.

Advanced usage

To fully utilize the Dormant functionality available in FriendUP, you need to use either a FriendUP Shell, or a programmers editor. Thankfully, both are included in the FriendUP Workspace. Using the Dormant functions using a Shell or a Shell script parser (one is built into our FriendDOS Shell), you can program your own functionality.

Using other apps

To show you more interesting use cases, fire up FriendCreate, like this, still using the FriendDOS Shell application:

launch FriendCreate
enter FriendCreate
SelectAll
Insert Hello there!
SelectAll
Insert Some other text...
leave

You have now used the Shell to write some lines in the FriendCreate text editor. For even more advanced usage of Dormant and the Shell, please take a look at the documentation of FriendDOS.

Getting more information

Each application that has exposed its functionality through Dormant will have a dedicated chapter on the subject. Each application installed in your Workspace populates the central documentation directory located here:

System:Documentation/Applications/

As for the system functions, please look under FriendDOS and Workspace respectively, in the System:Documentation/ directories.
